# 카카오톡 링크 문제 최종 확인 가이드

## 현재 상황
- ✅ 코드: Vercel URL 사용 확인됨
- ✅ 콘솔 로그: Vercel URL 출력 확인됨
- ❌ 카카오톡 링크 클릭: GitHub로 이동

## 핵심 문제
카카오톡이 링크를 **크롤링**할 때 다른 정보를 보고 있을 가능성이 높습니다.

## 확인 방법

### 1. 카카오톡 개발자 콘솔 - 링크 검증 도구 (가장 중요)

1. **https://developers.kakao.com** 접속
2. 내 애플리케이션 선택
3. **도구** → **링크 검증** 메뉴
4. URL 입력: `https://wedding-card-eight-kappa.vercel.app/`
5. **검증** 버튼 클릭
6. **결과 확인:**
   - `og:url` 값이 무엇인지 확인
   - `og:image` 값이 무엇인지 확인
   - 만약 GitHub URL이 나온다면 → **캐시 문제**
   - 만약 Vercel URL이 나온다면 → **다른 설정 문제**

### 2. 카카오톡이 실제로 크롤링하는 URL 확인

카카오톡은 링크를 크롤링할 때:
1. 전달된 URL을 크롤링
2. Open Graph 메타 태그 확인
3. `og:url`이 있으면 그 URL로 리다이렉트

**확인 방법:**
- 링크 검증 도구에서 `og:url` 값 확인
- 만약 `og:url`이 GitHub URL이면 → `index.html`의 메타 태그 확인

### 3. Vercel 사이트에서 실제 메타 태그 확인

1. 브라우저에서 `https://wedding-card-eight-kappa.vercel.app/` 접속
2. 개발자 도구(F12) → Elements 탭
3. `<head>` 태그 내부의 `<meta property="og:url">` 확인
4. 실제 값이 무엇인지 확인

### 4. 카카오톡 개발자 콘솔 - 플랫폼 설정 재확인

1. **앱 설정** → **플랫폼**
2. **Web 플랫폼** 섹션
3. **사이트 도메인** 목록:
   - `gusehd66.github.io`가 **완전히 삭제**되었는지 확인
   - `wedding-card-eight-kappa.vercel.app`가 **추가**되어 있는지 확인
4. **저장** 버튼 클릭

### 5. 카카오톡 앱 완전 초기화

1. 카카오톡 앱 완전 종료
2. 앱 데이터 삭제 (선택사항)
3. 앱 재시작
4. **새로운 링크로 공유** (이전 링크는 삭제)

## 가능한 원인

### 원인 1: 카카오톡 캐시
- 카카오톡이 이전에 크롤링한 정보를 캐시하고 있음
- 해결: 링크 검증 도구에서 캐시 삭제 + 24시간 대기

### 원인 2: Open Graph 메타 태그
- Vercel 사이트의 실제 HTML에서 `og:url`이 GitHub URL일 수 있음
- 해결: `index.html`의 메타 태그 확인 및 수정

### 원인 3: 카카오톡 플랫폼 설정
- Web 플랫폼의 사이트 도메인에 GitHub가 등록되어 있음
- 해결: GitHub 도메인 삭제, Vercel 도메인 추가

### 원인 4: 리다이렉트
- Vercel 사이트에서 GitHub로 리다이렉트가 발생할 수 있음
- 해결: 네트워크 탭에서 리다이렉트 확인

## 즉시 확인할 것

1. **카카오톡 개발자 콘솔** → **링크 검증** → Vercel URL 검증
2. **브라우저** → Vercel 사이트 접속 → Elements 탭 → `og:url` 메타 태그 확인
3. **카카오톡 개발자 콘솔** → **플랫폼** → Web 플랫폼 사이트 도메인 확인
