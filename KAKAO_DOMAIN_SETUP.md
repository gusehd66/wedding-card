# 카카오톡 개발자 콘솔 도메인 설정 가이드

## 문제 원인

코드상으로는 Vercel URL을 사용하고 있지만, 카카오톡이 GitHub 링크로 공유되는 이유는 **카카오톡 개발자 콘솔의 플랫폼 설정** 때문일 수 있습니다.

## 해결 방법

### 1. 카카오톡 개발자 콘솔 접속

1. https://developers.kakao.com 접속
2. 로그인 후 내 애플리케이션 선택 (JavaScript Key: `51ce90a5a85c4e81932e03038c1559be`)

### 2. 플랫폼 설정 확인 및 수정

**중요: Web 플랫폼 설정**

1. **앱 설정** → **플랫폼** 메뉴 클릭
2. **Web 플랫폼** 섹션 확인
3. **사이트 도메인** 설정:

   - 기존: `gusehd66.github.io` (있다면 삭제)
   - 추가: `wedding-card-eight-kappa.vercel.app`
   - 또는 `*.vercel.app` (와일드카드 사용 가능)

4. **저장** 버튼 클릭

### 3. 링크 검증 도구 사용

1. **도구** → **링크 검증** 메뉴 클릭
2. URL 입력: `https://wedding-card-eight-kappa.vercel.app/`
3. **검증** 버튼 클릭
4. Open Graph 정보 확인:
   - `og:url`이 Vercel URL인지 확인
   - `og:image`가 Vercel URL인지 확인
5. **캐시 삭제** 버튼 클릭 (있다면)

### 4. 앱 키 확인

1. **앱 설정** → **앱 키** 메뉴
2. **JavaScript 키** 확인: `51ce90a5a85c4e81932e03038c1559be`
3. **플랫폼** 탭에서 Web 플랫폼이 활성화되어 있는지 확인

### 5. 추가 확인사항

**카카오톡이 링크를 크롤링할 때:**

- Open Graph 메타 태그의 `og:url`을 확인
- 현재 `index.html`에는 Vercel URL이 설정되어 있음
- 하지만 카카오톡이 이전에 캐시한 정보를 사용할 수 있음

**Vercel 도메인 설정:**

- Vercel 대시보드의 Domains 설정은 카카오톡과 직접적인 관련이 없음
- 카카오톡은 코드에서 전달된 URL과 Open Graph 메타 태그를 사용

## 즉시 해결 방법

1. **카카오톡 개발자 콘솔** → **플랫폼** → **Web** → 사이트 도메인에 `wedding-card-eight-kappa.vercel.app` 추가
2. **링크 검증 도구**에서 Vercel URL 검증 및 캐시 삭제
3. **새로운 링크로 다시 공유** (이전 링크는 삭제)
4. **카카오톡 앱 재시작**

## 참고

카카오톡은 링크 정보를 캐시하므로, 설정 변경 후에도 즉시 반영되지 않을 수 있습니다.
캐시 삭제 후 몇 시간에서 하루 정도 기다려야 할 수도 있습니다.
